So our first experience with numbers is as analogies. We can develop some basic notions of order (the “2-situations” all contain “1-situations” inside them, but not vice-versa). We can also define addition as a combining operation, and multiplication as “bulk replacement”. Finally, we might recognize the “nothing situation” which contains no objects; we've discovered 0.

Natural numbers

In treating numbers as analogies, we find that we can always add another number to our collection. We also have a smallest number: 0. These two properties define the natural numbers, N\mathbb{N}N:

0∈Nn∈N  ⟹  suc(n)∈N\begin{align*} 0 &\in \mathbb{N} \\ n \in \mathbb{N} \implies \mathrm{suc}(n) &\in \mathbb{N} \end{align*}0n∈N⟹suc(n)​∈N∈N​

That is, there's some natural number called 000, and given any natural number nnn, there's another natural number suc(n)\mathrm{suc}(n)suc(n). We can use these to recursively build up the first few natural numbers:

0∈Nsuc(0)∈Nsuc(suc(0))∈Nsuc(suc(suc(0)))∈N⋯\begin{align*} 0 &\in \mathbb{N} \\ \mathrm{suc}(0) &\in \mathbb{N} \\ \mathrm{suc}(\mathrm{suc}(0)) &\in \mathbb{N} \\ \mathrm{suc}(\mathrm{suc}(\mathrm{suc}(0))) &\in \mathbb{N} \\ &\cdots \end{align*}0suc(0)suc(suc(0))suc(suc(suc(0)))​∈N∈N∈N∈N⋯​

We can define addition by:

0+n=nsuc(m)+n=suc(m+n)\begin{align*} 0 + n &= n \\ \mathrm{suc}(m) + n &= \mathrm{suc}(m + n) \end{align*}0+nsuc(m)+n​=n=suc(m+n)​

This definition looks circular, but it's not: if the first number is 000, it produces an answer; otherwise it reduces to a smaller addition problem. Here's 2+32 + 32+3:

suc(suc(0))+suc(suc(suc(0)))=suc(suc(0)+suc(suc(suc(0))))=suc(suc(0+suc(suc(suc(0)))))=suc(suc(suc(suc(suc(0)))))\begin{align*} \mathrm{suc}(\mathrm{suc}(0)) + \mathrm{suc}(\mathrm{suc}(\mathrm{suc}(0))) &= \mathrm{suc}(\mathrm{suc}(0) + \mathrm{suc}(\mathrm{suc}(\mathrm{suc}(0)))) \\ &= \mathrm{suc}(\mathrm{suc}(0 + \mathrm{suc}(\mathrm{suc}(\mathrm{suc}(0))))) \\ &= \mathrm{suc}(\mathrm{suc}(\mathrm{suc}(\mathrm{suc}(\mathrm{suc}(0))))) \end{align*}suc(suc(0))+suc(suc(suc(0)))​=suc(suc(0)+suc(suc(suc(0))))=suc(suc(0+suc(suc(suc(0)))))=suc(suc(suc(suc(suc(0)))))​

We can define multiplication in a similar fashion:

0⋅n=0suc(m)⋅n=n+(m⋅n)\begin{align*} 0 \cdot n &= 0 \\ \mathrm{suc}(m) \cdot n &= n + (m \cdot n) \end{align*}0⋅nsuc(m)⋅n​=0=n+(m⋅n)​

We can prove that addition is associative:

(0+b)+c=b+c=0+(b+c)(suc(a)+b)+c=suc(a+b)+c=suc((a+b)+c)=suc(a+(b+c))[Induction hypothesis]=suc(a)+(b+c)\begin{align*} (0 + b) + c &= b + c \\ &= 0 + (b + c) \\ (\mathrm{suc}(a) + b) + c &= \mathrm{suc}(a + b) + c \\ &= \mathrm{suc}((a + b) + c) \\ &= \mathrm{suc}(a + (b + c)) &\textrm{[Induction hypothesis]} \\ &= \mathrm{suc}(a) + (b + c) \end{align*}(0+b)+c(suc(a)+b)+c​=b+c=0+(b+c)=suc(a+b)+c=suc((a+b)+c)=suc(a+(b+c))=suc(a)+(b+c)​[Induction hypothesis]​

N\mathbb{N}N is a “commutative monoid” under both addition and multiplication. This means that both addition and multiplication are associative, commutative, and have an identity. We can prove all of these facts. Why is N\mathbb{N}N not a commutative monoid under exponentiation?

Integers

The integers are an extension of the natural numbers, in both an informal and technical sense. We begin by defining the set Z=N×N\mathrm{Z} = \mathbb{N} \times \mathbb{N}Z=N×N. That is, Z\mathrm{Z}Z is the set of pairs of natural numbers. We think of the elements of Z\mathrm{Z}Z, say (a,b)(a, b)(a,b) as representing the difference a−ba - ba−b. This allows us to represent negative numbers.

When defining N\mathbb{N}N there was only one way to represent each number. Now there are many ways (an infinite amount) to represent the same number. We capture this idea with an equivalence relation:

(a,b)∼(c,d)⟺a+d=b+c(a, b) \sim (c, d) \Longleftrightarrow a + d = b + c(a,b)∼(c,d)⟺a+d=b+c

So (1,3)∼(2,4)(1, 3) \sim (2, 4)(1,3)∼(2,4) and (5,4)∼(1,0)(5, 4) \sim (1, 0)(5,4)∼(1,0).

We can define addition on Z\mathrm{Z}Z by

(a,b)+(c,d)=(a+c,b+d)(a, b) + (c, d) = (a + c, b + d)(a,b)+(c,d)=(a+c,b+d)

Can we use this to define addition on Z\mathbb{Z}Z? Yes and no. The elements of Z\mathbb{Z}Z are entire equivalence classes (subsets) of Z\mathrm{Z}Z. In this cases, we could select some “canonical representative” of each equivalence class, which would allow us to define addition on Z\mathbb{Z}Z, but we won't always be able to do this. Rational numbers

The construction of the rationals will look familiar. We begin by defining a set

Q={(x,y)∈Z×Z∣y≠0}\mathrm{Q} = \left\{ (x, y) \in \mathbb{Z} \times \mathbb{Z} \mid y \neq 0 \right\}Q={(x,y)∈Z×Z∣y=0}

That is, Q\mathrm{Q}Q is the set of pairs of integers, where the second element isn't 000. Here we're thinking of each pair (x,y)(x, y)(x,y) as the quotient xy\frac{x}{y}yx​. We define an equivalence relation ∼\sim∼ by

(x,y)∼(z,w)⟺xw=yz(x, y) \sim (z, w) \Longleftrightarrow xw = yz(x,y)∼(z,w)⟺xw=yz

We define addition on Q\mathrm{Q}Q:

(x,y)+(z,w)=(xw+yz,yw)(x, y) + (z, w) = (xw + yz, yw)(x,y)+(z,w)=(xw+yz,yw)

Real numbers

The ratio of the length of the diagonal of a square to its side is not expressible as a rational number. Yet both of these lengths are finite, and fairly similar in length. It feels like this ratio ought to be expressible as a number.

This should look familiar by now. Define

R={σ∈N→Q∣σ is Cauchy}\mathrm{R} = \left\{ \sigma \in \mathbb{N} \to \mathbb{Q} \mid \sigma \textrm{ is Cauchy} \right\}R={σ∈N→Q∣σ is Cauchy}

A sequence σ\sigmaσ is “Cauchy” if its terms get arbitrarily close to each other at some point. Specifically, iff

∀B∈N,∃N∈N:m,n≥N  ⟹  ∣σ(m)−σ(n)∣<1B\forall B \in \mathbb{N}, \exists N \in \mathbb{N} : m, n \geq N \implies |\sigma(m) - \sigma(n)| < \frac{1}{B}∀B∈N,∃N∈N:m,n≥N⟹∣σ(m)−σ(n)∣<B1​

Define ∼\sim∼ by

σ∼τ⟺(σ−τ)→0\sigma \sim \tau \Longleftrightarrow (\sigma - \tau) \to 0σ∼τ⟺(σ−τ)→0

where we've introduced a new idea: →0\to 0→0. A sequence σ\sigmaσ converges to 000 iff

∀B∈N,∃N∈N:n≥N  ⟹  ∣σ(n)∣<1B\forall B \in \mathbb{N}, \exists N \in \mathbb{N} : n \geq N \implies |\sigma(n)| < \frac{1}{B}∀B∈N,∃N∈N:n≥N⟹∣σ(n)∣<B1​

To recap: we've defined a new set R\mathrm{R}R as the set of Cauchy sequences of rational numbers, and said that two elements of R\mathrm{R}R are equivalent if their difference converges to 000.

Then

R=R /∼\mathbb{R} = \mathrm{R} \thinspace / \simR=R/∼

R\mathbb{R}R is Cauchy complete: every Cauchy sequence in R\mathbb{R}R has a limit.
